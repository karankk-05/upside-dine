Upside Dine - Person A Work (Users/Auth) Summary
================================================

Date: 2026-03-15
Scope: Implement Person A responsibilities (Users/Auth) and wire into backend.

What was implemented
--------------------
1. Custom user and role system
   - Custom User model (email login) with roles and verification flags
   - Role model with predefined roles
   - Student and Staff profile models
   - MessAccount model
   - UserToken model to store refresh tokens

2. Auth and profile APIs
   - Register with OTP email
   - Verify OTP
   - Login (JWT access + refresh)
   - Refresh access token
   - Logout (blacklist refresh JTI)
   - Forgot password (OTP)
   - Reset password (OTP)
   - Current user profile (GET/PATCH)
   - Mess account view

3. OTP and rate limiting
   - OTP generation and verification via Redis cache
   - OTP attempt rate limit
   - API rate limit middleware (per-IP basic throttling)

4. JWT authentication
   - Custom JWTAuthentication that checks Redis blacklist

5. Admin and forms
   - Custom admin registration for user-related models
   - Custom admin forms for the User model

6. Wiring and settings
   - Added app to INSTALLED_APPS
   - Set AUTH_USER_MODEL
   - Registered routes in config/urls.py
   - Switched REST framework auth to custom JWTAuthentication
   - Added email settings in config/settings.py

7. Basic test
   - Smoke test to ensure non-@iitk email registration fails
